03 — Recursion / Project Lifecycle

How Project N spawns Project N+1

About this diagram

The Recursion / Project Lifecycle Map shows the core mechanism by which the NovoClaw Recursive Agent Factory scales: each successful project funds and spawns the next one. A new project is only provisioned when two gate conditions are met — seed capital is fully repaid and the project sustains at least 25% monthly ROI. When both gates pass, the Investor Agent allocates 85% of net profit to provision a new PaperClip container, seed a fresh G2 agent, and transfer knowledge from the parent project. The new project inherits 95% of the shared operating stack, reducing spin-up time and cost.

The recursive loop at the bottom illustrates that once Project N+1 hits the same trigger conditions, it will spawn Project N+2 — creating a self-replicating chain of autonomous revenue projects, each governed by the same kill-switch and rollback policies.

Kill-switches exist at three levels: automated (ROI threshold), systemic (container deallocation and capital return), and human (Kenny retains override authority to pause or terminate any project at any time). This ensures the factory never runs unchecked.
