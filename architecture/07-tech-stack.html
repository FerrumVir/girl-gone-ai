07 — Tech Stack Diagram

Services, APIs, auth surfaces, and host infrastructure

About this diagram

Full tech stack from user-facing pages down to host infrastructure. The stack is organized in four layers showing the flow from what visitors see (girlgone.ai, Gumroad storefront) through the API and integration layer, the Paperclip agent platform that orchestrates everything, and the underlying Linux host and GitHub deployment pipeline.

  • Layer 1 (User-Facing) — Three entry points: the static site on GitHub Pages, the Gumroad storefront for purchases, and the Bubble.io dashboard for Jaci's task management (currently partially broken).
  • Layer 2 (APIs & Integrations) — Gumroad API for product management (10 products/day rate limit), HeyGen for avatar video generation (currently returning 401), Meta Pixel and Google Ads tags for analytics, Dropbox for file storage, and Stripe payments routed through Gumroad.
  • Layer 3 (Agent Platform) — Paperclip server on localhost:3100, Vite dev UI, the Claude CLI agent runtime (claude-opus-4-6), and a PostgreSQL state database. The agent runtime connects outbound to Gumroad API, HeyGen API, and GitHub.
  • Layer 4 (Host Infrastructure) — Ubuntu Linux with NVIDIA GPU running Node.js/pnpm, Python 3, Git, and the gh CLI. GitHub houses the FerrumVir/girl-gone-ai repo with GitHub Actions auto-deploying to GitHub Pages on push to main.
  • Auth Surface — Five server-side credentials (Gumroad API token, HeyGen API key, GitHub token, Paperclip JWT) and two client-side tags (Meta Pixel, Google Ads). One credential is currently broken (HeyGen 401).
