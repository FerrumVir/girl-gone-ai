06 — Data & Asset Topology

Where assets live, source of truth, and sync rules

About this diagram

Assets are primarily stored in the girlgone.ai GitHub Pages repo, which serves as the single source of truth for all product content, cover art, thumbnails, social banners, and delivery pages. External stores (Gumroad, Bubble.io, Dropbox, HeyGen) receive data via sync pipelines or manual push. The repo currently contains 160+ product pages, four generations of cover art directories, and platform-specific banner sets for Instagram, TikTok, X/Twitter, Facebook, YouTube, LinkedIn, and Pinterest.

Convergence Target

Multiple cover directories (covers, covers2, covers3, covers4) should converge into a single /covers/ directory with a consistent naming convention. This simplifies asset generation pipelines, reduces confusion about which generation of cover art is canonical, and makes automated thumbnail generation straightforward. The target pipeline is linear: content creation, asset generation, publication to girlgone.ai, syndication to Gumroad, and customer delivery.

Dropbox Smart Sync is a workaround — online-only mode means assets are not fully local and depend on network availability. Bubble.io dashboard sync is partially broken (JAC-482). HeyGen API currently returns 401 auth failures, blocking video asset generation.
